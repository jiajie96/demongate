## Project Overview

Hellgate Defenders is a tower defense game built with Godot 4.6 (GDScript). The player defends Hell's Core against 20 waves of divine warriors using 4 tower types, with gambling mechanics (Devil's Dice, Demonic Pacts) and Pandora's Relics.

## Architecture

The game uses Godot autoloads (singletons) as the backbone. There are no custom classes or node hierarchies — all game entities (towers, enemies, projectiles) are plain Dictionaries managed in Arrays on the game manager.

**Autoloads** (registered in `project.godot`, accessed globally by name):
- `Config` → `scripts/autoload/game_config.gd` — Pure data: tower/enemy/wave stats, dice outcomes, pact pool, relic loot table, map path, grid math helpers. All balance tuning happens here.
- `GM` → `scripts/autoload/game_manager.gd` — All game state and logic: economy (sins currency), entity CRUD, combat/targeting, wave spawning, gambling systems (dice/pacts/relics), effects/notifications. Owns the arrays `towers`, `enemies`, `projectiles`, `effects`.
- `Audio` → `scripts/autoload/audio_manager.gd` — Procedurally generated sound effects and music (no audio files). Call `Audio.play_sfx()` / `Audio.play_shoot()`.

**Scene scripts:**
- `scripts/game_world.gd` (Node2D) — Game loop driver (`_process` calls GM.update_*) and all 2D rendering via `_draw()`. Also handles mouse input for tower placement.
- `scripts/hud.gd` (CanvasLayer) — Builds all UI programmatically (no .tscn UI), handles button callbacks for tower selection, upgrades, selling, dice, pacts.
- `scripts/main.gd` — Just draws a border around the game area.

**Key patterns:**
- Entities are Dictionaries, not objects. Tower dict has keys like `id`, `type`, `level`, `damage`, `cooldown`, `is_disabled`. Enemy dict has `hp`, `path_index`, `slow_amount`, etc.
- Game phases: `"menu"` → `"playing"` → `"gameover"` or `"victory"` (stored in `GM.phase`).
- Currency is "Sins". Earned from kills and wave completion, spent on towers.
- Pacts are offered every 5 waves (`Config.PACT_EVERY`), presenting benefit/cost tradeoffs.
- Tower targeting picks the enemy with highest `path_index` (furthest along the path) within range.

## Tests

`tests/test_runner.gd` contains 310+ tests using a simple `_assert`/`_assert_eq` framework (no external test library). Tests cover config integrity, map validity, economy, tower mechanics, enemy behavior, combat math, wave definitions, slow mechanics, and gambling data. Add new test groups as `_run_*_tests()` methods called from `_ready()`.
